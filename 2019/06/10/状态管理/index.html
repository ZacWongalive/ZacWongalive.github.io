状态管理

  1. 什么是状态管理？

    • 将浏览器和web服务器之间的多次交互几位一个整体来处理，并且将多次交互所涉及的数据（即状态）保存下来

  2. 进行状态管理的方式有几种？

    1. cookie(在客户端管理用户的状态)
    2. session（在服务段管理用户的状态）

cookie

  1. 什么是cookie?

    • 是服务器临时保存在浏览器端的少量的数据。

  2. cookie的工作原理

    • 当浏览器访问服务器时，服务器可以将少量数据以ser-cookie消息头的方式发送给浏览器，浏览器会将这些数据临时保存寄来。 当浏览器再次访问服务器时，会将之前保存的这些数据以cookie消息头的方式发送给服务器。

  3. 如何添加cookie?

    Cookie c = new Cookie(String name , String value);

    `response.addCookie(c);`

  4. cookie的读取

    Cookie[] rquest.getCookies()

String  Cookie.getName()//获取cookie名字
String cookie.getValue()//获取cookie的值

  5. 关于cookie添加和读取的编码问题

    1. cookie只能保存合法的ascii字符，对于中文，需要将其转化成相应的ascii字符表示形式。

      URLEncoder.encode(s,”utf-8”)
      URLDecoder.decode(s,”utf-8”)
      注：添加cookie时，建议最好统一编码处理

  6. cookie的生存时间问题

    1. 缺省情况下，浏览器会把cookie保存在内存中，只要浏览器不关闭，cookie就会一直存在，浏览器关闭，cookie就会销毁。

      setMaxAge(int second)
      注意：参数是秒

      second > 0 浏览器会将cookie保存在硬盘上，如果超过了指定的时间，则cookie会被删除

      second < 0 默认缺省值

      second = 0 删除cookie

      就是这样咯

  1. 怎么修改cookie?

    1. 服务器向浏览器再发送一个同名的cookie，那么原先浏览器中保留的cookie会被覆盖。

  2. 如何删除一个cookie?

    在服务器端创建好一个和客户端同名的

    cookie c = new cookie(“cart”,””);cookie名

    设置该cookie的最大生存时间

    c.setMaxAge(0)

    向浏览器发送cookie

    response.addCookie(c)

  3. cookie的路径问题

    1. 什么是cookie的路径问题？

      • 浏览器再向服务器上的某个地址发送请求时，会比较该地址是否符合cookie的路径的要求，只有符合条件的cookie才能被发送出去

    2. cookie的默认路径

      • 等于添加该cookie的组件(servlet/jsp)的路径
      1. 匹配原则
        • 请求地址必须等于cookie的路径，或是其子路径

  4. cookie的限制

    • 不安全，如果有名干数据必须以cookie的方式来保存，必须加密。
    • 可以被用户禁止
    • cookie只能保存少量的数据(4k)
    • 浏览器能保存的cookie的数量也有限(大概300个左右)
    • 只能保存字符串

session(会话)

  1. 什么是session?

    • 服务器端为维护状态而创建的一个特殊对象

  2. 工作原理

    • 浏览器访问服务器时，服务器会创建一个session对象（该对象与一个唯一的id，称之为sessionid）,服务器会将sessionid以cookie的方式发送给浏览器，然后当浏览器再次访问服务器，会将sessionid以cookie的方式发送过来，服务器通过sessionid找到对应的session对象

  3. 如何获取session对象？

    方式1

    HttpSessoin s = request.getSession(true/false);

    true:先查看请求当中有没有sessionid，如果没有，创建一个session对象。如果有sessionid,则依据sessionid查找对应的session对象。

    false:先查看请求当中有没有sessionid，如果没有则返回null。如果有sessionid则会在服务器端查找session对象如果找不到session对象则会返回Null。

    方式2

    HttpSession s = request.getSession()

    等价于方式1中的true情况request.getSession(true)

  4. session当中常用的方法

    • String getId() 获取sessionid
    • setAttribute(String a ,object b) 绑定数据 用来设置session值的，sessionName是名称，object是你要保存的对象。
    • getAttribute(String a)该方法有可能返回null用来得到对应名称的session值，即得到object对象，注意需要进行类型转换！
    • removeAtrribute(String name)

  5. session超时

    1. 什么是session超时
      • 服务器会将空闲时间过长的session对象删除掉
    2. 缺省的超时。时间
      • 服务器缺省的时间限制一般是30分钟
    3. 设置超时时间的方法
      • session.setInActiveInterval(int seconds);

  6. session的删除

    1. session对象.invalidate()
