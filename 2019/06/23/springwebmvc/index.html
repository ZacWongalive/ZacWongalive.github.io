spring web mvc小案例

  1. 流程

    /hello.do –> spring web mvc –> /WEB-INF/hello.jsp
    /hello.do–>DispatcherServlet(配置)–>HanlderMapping（配置）–>helloController（配置+编写）–>ViewResolver（配置）–>/WEB-INF/hello.jsp（编写）

  2. 具体操作,详细看传智springmvc01

    1. 搭建spring web mvc 开发环境

      1. 引入 ioc,mvc的jar包
      2. src下添加applicationContext.xml

    2. 编写HelloController实现

      1. 规则：实现controller接口

    3. 配置DispatcherServlet

      • 配置web.xml配置 那么为什么要在web.xml当中配置DispatcherServlet的同时还要加上applicationContext.xml初始化参数？那是因为这样做会使得两个文件相关联，applicationContext.xml中可以定义Handlermapping,controller,viewsolver.当请求过来了服务器会创建DispatcherServlet对象，这样这个对象就可以通过applicationContext.xml当中的组件了

    4. 配置applicationContext.xml当中的组件

通过注解配置applicationContext.xml

  1. 对于hadlemapping 详细看p7 传智spingmvc04

    1. 使用RequestMappingHanlderMapping组件@RequestMapping("/login.do")该标记写在controller业务方法前 并且在applicationContext.xml文件当中写上<mvc:annotation-driven/>就可以直接省去以往的handlemapping的bean标签

  2. 对于Controller

    1. 取消了实现Controller接口及方法约定,可以允许程序员按照需要灵活定义业务方法如：public ModelAndView或者String 方法名(根据参数定义需要传入参数，request,session,response){} 定义返回值为string的话就意味着这个controller虽然只会返回viewName(视图名字)，springmvc底层还是会将返回的字符串封装到modelandview对象中传给viewreslover 但是这样使用的条件：Controller需要扫描到Spring容器，而且必须使用@Controller组件标注类

  3. 通过注解的方式处理多个请求（小案例）

    1. 详细看传智springmvc05

  4. Controller如何接收请求参数

    1. 利用HttpServletRequest
    2. 利用业务方法参数
      1. 参数名与请求参数ky保持一致
      2. 参数名和请求参数不一致在方法参数那写上@RequestParam("key")
    3. 利用实体对象当方法参数
      1. 第三条的出现，就是为了解决将来会出现的很多个参数传入的情况,容器会自动依据User类里头有没有与传入参数相同名字的属性而传入不传入User类实例。
        -
    4. 使用建议：后面两种方式都要注意，必须得保证请求传来的参数必须为String类型的

  5. Controller如何向响应jsp传值

    1. 利用HttpServletRequest
    2. 利用ModelAndView做返回值
    3. 利用ModelMap方法参数详细看springmvc05 logincontroller.java

  6. Controller如何获取并使用session对象？

    1. 直接从controller自定义方法参数中request的方法获取。
    2. 在controller自定义方法参数添加一个hpptSession 参数。

springmvc工作流程

  1. 浏览器发出http请求到dispatcherSevlet主控制器
  2. 主控制器会调用handlemapping组件依据的是请求和controller的映射关系
  3. 执行controller方法然后将返回结果给viewReslover.
  4. viewResolver会依据返回结果去找jsp，将模型数据给jsp
  5. jsp生成响应结果，给浏览器显示

解决springmvc项目当中中文接受乱码的问题

  1. 出现乱码的两种可能
  2. 对于第一种乱码产生的原因&解决方法
    • 产生原因：数据提交过来还是存在request对象当中，但是如果在不指明编码格式的情况下，服务器会按照ios8的编码格式来操作，springmvc框架只是封装底层的这些操作，这样默认还是按照ios编码操作
    • 解决办法1：在项目前面加上过滤器Filter
    • 解决办法2：在项目的web.xml文件前面加上过滤器配置（springmvc自带的类包）
  3. 对于第二种乱码的问题详细看Servelt文章

springmvc项目中如何处理异常

  1. 一般都是controller当中的方法出错了
    1. 全局异常处理方式第一种
      • 在applicationContext.xml文件当中添加springmvc自带的异常处理标签（全局异常处理） 详细看传智rekone
      • springmvc异常处理的过程
        • 当执行到第三步的时候controller抛出异常，抛给hadlemapping，然后再抛给dispatcherservlet，然后dispatcherservlet会看applicationContext.xml文件当中是否有处理Controller异常的标签，有的话就会依据异常标签当中的<prop key="异常类型">/WEB-INF路径下的错误处理页面名字</prop>抛给viewreslover去找异常处理jsp页面
    2. 局部异常处理方式
      1. 在controller文件当中加一个局部异常处理
        • 只需要在controller当中再定义一个方法，头上加一个ExceptionHandler
        • 只要异常被这个局部异常处理了，容器就不会调用全局异常
    3. 全局异常处理方式第二种方式
      1. 写一个类实现HandlerExceptionResolver，然后将信息写入到文件当中，然后再到applicationContext.xml配置一个标签
